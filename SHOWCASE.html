DevOps Debugging & Stabilizing Journal

Real-world infrastructure automation involves solving highly complex cross-vm networking, process management, and permissions issues. This journal outlines 12 real engineering bugs encountered, investigated, and systematically patched to build this robust, production-grade system.

ISSUE 01 NAT Instance Silently Dropping Packets
Symptom
Private subnet instances (VM2, VM3) completely unable to reach the internet or download packages. `apt-get update` hangs indefinitely.
Root Cause Analysis
AWS EC2 instances enforce source_dest_check = true by default. This drops any packet traversing the instance whose destination IP does not match the instance's own IP. A NAT instance must act as a router and forward packets for other IPs.
Terraform Resolution (nat_instance.tf)
resource "aws_instance" "nat" {
  # ...
  source_dest_check           = false # CRITICAL: Disable check for NAT forwarding
  user_data                   = <<-EOF
    #!/bin/bash
    echo 1 > /proc/sys/net/ipv4/ip_forward
    DEFAULT_INTERFACE=$(ip route show | awk '/default/ {print $5}')
    iptables -t nat -A POSTROUTING -o "$DEFAULT_INTERFACE" -j MASQUERADE
  EOF
}
ISSUE 02 unattended-upgrades Holding dpkg Lock
Symptom
Ansible common role fails during initial boot setup: E: Could not get lock /var/lib/dpkg/lock-frontend - open (11: Resource temporarily unavailable).
Root Cause Analysis
Ubuntu cloud images trigger background automatic updates via the unattended-upgrades service shortly after network initialization. This locks the dpkg database, causing concurrent Ansible apt tasks to crash.
Ansible Task Fix (roles/common/tasks/main.yml)
# Wait for apt lock to release safely before executing packages setup
- name: Wait for apt lock to be released
  shell: |
    while fuser /var/lib/dpkg/lock-frontend /var/lib/dpkg/lock /var/cache/apt/archives/lock >/dev/null 2>&1; do
      echo "Waiting for apt lock..."
      sleep 5
    done
  changed_when: false
  ignore_errors: yes
ISSUE 03 PyTorch Wheel Extraction Disk Exhaustion
Symptom
Inference Worker setup halts partway during pip installs. SSH throws No space left on device warnings.
Root Cause Analysis
Standard CUDA PyTorch wheels exceed 2.2GB. Pip cached wheels and extracted files in /tmp, completely filling the default 8GB EBS boot volume on standard AWS micro templates.
Architectural Resolution (instances.tf & worker)
# 1. Expanded base EBS root volumes in IaC (instances.tf)
root_block_device {
  volume_size = 20 # Provisioned 20GB root drive space
}

# 2. Abandoned massive GPU PyTorch wheels in favor of lightweight, CPU-optimized llama.cpp-python:
pip install llama-cpp-python --extra-index-url https://abetlen.github.io/llama-cpp-python/whl/cpu
ISSUE 04 WSL SSH ProxyCommand Path Resolutions
Symptom
Running Ansible playbooks from Windows terminal to manage private VMs fails with Connection refused or UNPROTECTED PRIVATE KEY FILE.
Root Cause Analysis
Windows SSH does not natively parse typical Linux proxy bastion jumps or relative PEM path formats. Additionally, files generated via Terraform on Windows carry open ACL security permissions, which SSH rejects.
PowerShell WSL Bridge Automations (deploy.ps1)
# Auto-detects WSL context, mirrors configuration key rules, and tunnels playbook execution
if ($UseWSLAnsible) {
    wsl -d Ubuntu sh -c "cp ../terraform/iii-key.pem /tmp/iii-key.pem && chmod 400 /tmp/iii-key.pem"
    (Get-Content inventory.ini) -replace '\.\.\/terraform\/iii-key\.pem', '/tmp/iii-key.pem' | Out-File -Encoding ascii inventory_wsl.ini
    wsl -d Ubuntu sh -c "ansible-playbook -i inventory_wsl.ini playbook.yml"
}
ISSUE 05 systemd Startup Race Conditions
Symptom
Private worker processes crash and loop in `activating (start)` indefinitely on the initial Ansible deployment boot.
Root Cause Analysis
The 540MB arithmetic model must be downloaded from HuggingFace on first start. The systemd unit had the default `TimeoutStartSec=90` timer, causing systemd to assume the worker was hung and kill it repeatedly.
Service Cushioning (systemd/inference-worker.service)
[Service]
Type=simple
ExecStart=/opt/iii/workers/inference-worker/venv/bin/python inference_worker.py
# Allocate generous startup timeout for first-boot asset caching:
TimeoutStartSec=300
Restart=on-failure
RestartSec=10
ISSUE 06 WebSocket Public Route Interceptions
Symptom
completions request throws: `{"error": "No handler registered"}` even though worker services are fully active.
Root Cause Analysis
The worker's `III_URL` env was set to VM1's *public* IP. The VPC's firewall blocks external websocket ingress on port 49134. Workers must route over the *private* subnet boundary to the broker private IP.
Ansible Playbook Variable Bind (roles/inference-worker)
# Force worker daemon environmental configs to bind strictly to VM1 private interfaces:
[Service]
Environment=III_URL=ws://{{ engine_private_ip }}:49134
Environment=HF_HOME=/opt/iii/workers/inference-worker/cache
ISSUE 07 GGUF Model OOM Crash under Concurrent Load
Symptom
Inference worker functions successfully on isolated requests, but crashes when multiple request batches arrive. Dmesg shows OOM-Kills.
Root Cause Analysis
The model weights consume 290MB. During active token generation, llama.cpp context buffer allocations spike memory usage beyond the physical 1GB RAM of standard t3.micro tiers, triggering Linux OOM.
Virtual Memory Cushion (roles/common/tasks/main.yml)
# Configured a resilient 8GB system swap partition to back physical RAM blocks
- name: Create swap space allocations
  command: fallocate -l 8G /swapfile
- name: Mount swap
  command: swapon /swapfile
ISSUE 08 Logger Hook Method Failures
Symptom
Inference worker script exits immediately on boot with: `AttributeError: 'Logger' object has no attribute 'warning'`.
Root Cause Analysis
The proprietary `iii` SDK's built-in Logger class is a custom subclass wrapper. It exposes standard `.info()` and `.error()` calls, but completely lacks support for standard library `.warning()`.
Script Patch (quickstart/workers/inference-worker/inference_worker.py)
# Replaced non-standard warnings method mappings:
# logger.warning("Slow model load...") <-- CRASH
logger.info("[WARNING] Slow model load...")
ISSUE 09 Ansible Windows Carriage Return Failures
Symptom
Ansible in WSL fails to parse generated inventory file, throwing invalid character or token errors.
Root Cause Analysis
Python I/O file writing on Windows defaults to CRLF (`\r\n`) line endings. The embedded multiline SSH bastion proxy command gets corrupted when `\r` is injected into terminal shell paths.
PowerShell Post-Gen Sanitizer (deploy.ps1)
# Strips carriage returns and rewrites standard Unix-style inventory blocks:
(Get-Content inventory.ini -Raw).Replace("`r`n", "`n") | [System.IO.File]::WriteAllText("$PWD\inventory.ini", $_)
ISSUE 10 iii-Engine CLI Flag Deprecations
Symptom
`iii-engine.service` crashes on startup on VM1: `flag provided but not defined: --no-watch`.
Root Cause Analysis
The engine package's CLI interface underwent a breaking revision. The `--no-watch` flag was deprecated, causing modern daemon startups to reject it.
Service File Correction (systemd/iii-engine.service)
# Cleaned ExecStart syntax. Watching is disabled via production configuration instead:
[Service]
ExecStart=/usr/local/bin/iii start
Environment=NODE_ENV=production
ISSUE 11 Local PowerShell curl Single-Quote Parsers
Symptom
Executing standard single-quoted completions curl checks inside Windows PowerShell crashes with `Could not resolve host: role` errors.
Root Cause Analysis
PowerShell parses standard single quotes differently than Bash. It splits JSON arguments containing spaces into multiple array options, corrupting the payload boundaries.
Cross-Platform Wrapper (scripts/test-api.ps1)
# Bypassed using PowerShell's native Invoke-RestMethod block:
$body = @{ messages = @( @{ role = "user"; content = "15 * 3" } ) } | ConvertTo-Json
$response = Invoke-RestMethod -Uri "https://98.84.46.0/v1/chat/completions" -Method Post -Body $body -ContentType "application/json" -SkipCertificateCheck
ISSUE 12 Nginx 502 Bad Gateway during Engine Reboot
Symptom
Restarting `iii-engine` causes edge Nginx proxies to instantly throw `502 Bad Gateway` for up to 30 seconds.
Root Cause Analysis
Rebooting the central engine clears its in-memory WebSocket worker registry. It takes up to 6 seconds for workers in the private subnet to detect socket drop and reconnect. During this, Nginx has no active upstreams.
Nginx Resilience Config (nginx/iii-api.conf)
# Instruct Nginx proxy passes to dynamically retry connections during drop recoveries:
location /v1/ {
    proxy_pass http://127.0.0.1:3111;
    proxy_next_upstream error timeout http_502;
    proxy_next_upstream_tries 3;
    proxy_next_upstream_timeout 10s;
}

Production Hardening & 100x Scale-Up Architecture

Moving from a single developer sandbox environment to an enterprise-grade highly available ML platform requires deep enhancements across security boundaries, dynamic scaling mechanics, and model distribution.

Enterprise VPC Security Hardening Replace SSH password keys entirely with **AWS Systems Manager (SSM) Session Manager**. This allows operators to shell into private nodes directly through IAM profile validations, letting us block SSH Port 22 completely. Enable **VPC Flow Logs** to stream network access logs into CloudWatch for threat profiling.
High-Availability API Gateways In production, Nginx on a single instance represents a single point of failure (SPOF). Swap VM1 with an **AWS Application Load Balancer (ALB)** paired with an **Autoscaling Group (ASG)** of Nginx containers, distributed across multiple Availability Zones (Multi-AZ) with ACM-managed wildcard TLS certificates.
Dedicated Model Hosting (100x Scale) If the model scales 100x larger (e.g., a 70B parameter model requiring >140GB VRAM), commodity CPUs are unusable. Transition to AWS **g5.12xlarge** GPU instances (equipped with 4x NVIDIA A10G GPUs). Replace standard python workers with **Triton Inference Server** to manage dynamic batching and concurrent model context slots.
Caching & Message Queues Introduce an **ElastiCache (Redis)** cluster behind the API gateway VM. This handles instant retrieval of duplicate semantic prompts, reducing token overhead. Standardize worker backplanes from WebSockets to **AWS SQS** or **Apache Kafka** to handle queuing bursts gracefully without dropouts.
